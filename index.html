予算期間設定

会社情報画面に予算開始月を設定し、設定機能と表示画面で一貫した予算期間を適用する

課題: 現状、設定側（月次予算設定・費用設定）と表示側（予実対比・年間予実対比・月次集計・感度分析）で一貫した予算開始月の設定がない。全画面がカレンダー年（1月〜12月）をハードコードしている。
解決: 会社情報画面（会社スコープ）に「予算開始月」を追加し、全店舗の予算関連画面で参照する。
参考: 既存の ClosingFiscalYearSetting モデル（決算期設定）に同様の fiscal_year_start_month カラムが存在する。予算期間は決算期とは独立した設定として Company モデルに直接追加する。

2. Company モデル

class Company < ApplicationRecord
  validates :budget_start_month, inclusion: { in: 1..12 }, allow_nil: true

  # 指定年の予算年度の開始・終了year_monthを返す
  def budget_period(year)
    s = budget_start_month || 1
    if s == 1
      { from: year * 100 + 1, to: year * 100 + 12 }
    else
      { from: year * 100 + s, to: (year + 1) * 100 + (s - 1) }
    end
  end

  # 予算年度の月リスト（表示順）を返す
  def budget_months(year)
    s = budget_start_month || 1
    (0..11).map do |offset|
      m = ((s - 1 + offset) % 12) + 1
      y = offset < (13 - s) ? year : year + 1
      { year: y, month: m, year_month: y * 100 + m }
    end
  end
end

6. 参考: 既存 ClosingFiscalYearSetting

決算期設定との違い:
- ClosingFiscalYearSetting は独立テーブルで fiscal_year_start_month を持つ（デフォルト: 4月）
- 予算期間設定は companies.budget_start_month として直接追加（シンプルな1カラム追加）
- 将来的に両者を統合する場合は、決算期設定の fiscal_year_start_month と予算開始月を連動させることも検討可能
